# 0.1.6 ‚Äî 2026-06-08

## Changed

- **`fema_search_disasters`, `fema_get_public_assistance`, `fema_dataframe_query`, `fema_query_dataset`** ‚Äî error contracts corrected: input/domain validation failures now raise `ValidationError` (`-32007`) instead of `InvalidParams` (`-32602`). The `reason` field on the error payload is unchanged; only the JSON-RPC code changes.
- **`openfema-service`** ‚Äî OData 400 responses thrown via `validationError()` instead of `invalidParams()` for consistency with the tool contracts above.
- **`@cyanheads/mcp-ts-core` `^0.9.21 ‚Üí ^0.10.1`** ‚Äî picks up the DataCanvas SQL gate hardening (see Security below) and `.mcpbignore` root-anchor fix.

## Security

- **DataCanvas SQL gate** ‚Äî `fema_dataframe_query` inherits the upstream fix from `@cyanheads/mcp-ts-core` [#210](https://github.com/cyanheads/mcp-ts-core/issues/210): non-preparable statements (e.g. bare `PRAGMA`) now return the documented `non_select_statement` `ValidationError` instead of a raw `InternalError`, and `pragma_*` table functions are denied across both the pre-plan scan and the plan walk.
