# 0.1.7 — 2026-06-10

## Fixed

- **`fema_search_nfip` canvas staging** now pages the full matching result set (bounded by the 50k canvas cap) via `$skip`-based pagination, so canvas SQL aggregations see the complete dataset rather than only `limit` rows ([#5](https://github.com/cyanheads/fema-mcp-server/issues/5)). `input.limit` now controls the inline preview count only; `truncated` reflects cap-hit vs. exhaustion; `canvas_id`/`canvas_table` are omitted from the response when no canvas was staged (spilled=false), so models do not attempt describe/query calls on a non-existent canvas.
  - **OpenFEMA 503 HTML responses** (Akamai/Drupal rate-limit pages) now classify as retryable `ServiceUnavailable` instead of `unknown_dataset`.
  - **Canvas pagination** drops `$orderby` (upstream times out on sorted deep pagination; SQL `ORDER BY` runs at query time).
- **`fema_search_nfip` county code normalization**: 3-digit county codes (e.g. `201`) are automatically prepended with the state FIPS to produce a 5-digit code (e.g. `48201`) when `state` is provided; 5-digit codes pass through unchanged ([#6](https://github.com/cyanheads/fema-mcp-server/issues/6)). Output `claims[].county_code` description corrected with a 5-digit example.
- **`fema_query_dataset` unknown-dataset error message** no longer includes "HTML instead of JSON" transport detail ([#10](https://github.com/cyanheads/fema-mcp-server/issues/10)). `data.reason: "unknown_dataset"` contract unchanged.
