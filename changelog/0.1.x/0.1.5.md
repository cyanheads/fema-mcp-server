# 0.1.5 — 2026-06-08

## Fixed

- **`ia_declared`** mapped to `iaProgramDeclared` (legacy general-IA flag, false for major declarations since ~2012) instead of `ihProgramDeclared` (Individuals and Households Program). Fixes `fema_search_disasters`, `fema_get_disaster`, `fema://disaster/{number}` resource, and `fema_get_housing_assistance` description/recovery text. ([#7](https://github.com/cyanheads/fema-mcp-server/issues/7))
- **OR rollup** for program flags (`ia_declared`, `pa_declared`, `hm_declared`) across all designated-area rows. OpenFEMA returns one row per area and a flag may be false on some rows and true on others for the same disaster; the prior code read only the first row. All four touchpoints now accumulate with `rows.some(r => r.<flag> === true)`. ([#7](https://github.com/cyanheads/fema-mcp-server/issues/7))
- **`fema_search_disasters` pagination** applied `limit`/`offset` as `$top`/`$skip` on area-rows before deduplication, returning fewer declarations than requested and splitting a single disaster across pages. Tool now overfetches up to 10 000 area-rows, deduplicates to declaration-level, then slices `offset`/`limit` against the deduplicated list. `designated_area_count` is now the true area count for every disaster in scope. ([#8](https://github.com/cyanheads/fema-mcp-server/issues/8))
- **`RawDisasterDeclaration`** interface missing `ihProgramDeclared?: boolean` — added. ([#7](https://github.com/cyanheads/fema-mcp-server/issues/7))
